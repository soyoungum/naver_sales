# 스마트스토어 판매·마케팅 통합 대시보드 PRD

> **버전:** v1 · **작성일:** 2026-04-23 · **담당:** kangheon_jeon@fursys.com

---

## 1. 배경 & 목적

`smartstore_recording.py`로 매주 다운로드되는 스마트스토어 **판매분석(상품성과)** · **마케팅분석(전체채널)** Excel 파일을 한 화면에서 시각화·비교한다. 현재는 주차별 xlsx 파일이 `downloads/`에 누적될 뿐, 주차 간 비교나 채널-상품 연결 분석이 수기에 의존한다.

**해결하려는 문제**
- 주차별 매출/유입 추이를 한 번에 볼 수 없음
- 어떤 마케팅 채널이 실제 매출에 기여했는지 직관적으로 알기 어려움
- 시리즈(T60/T50/아이블 등)별 성과 비교를 매번 Excel로 수작업

---

## 2. 사용자 & 시나리오

| 사용자 | 주요 시나리오 |
|---|---|
| 운영 담당자 (1차 사용자) | 매주 월요일 아침, 지난주 데이터 다운로드 직후 대시보드 실행 → 주요 KPI 변동 확인 |
| 매니저 / 의사결정자 | 월말, 월간 트렌드 + 채널별 ROAS 검토 |

**핵심 사용 흐름**
1. `NAVER PROMOTION.cmd` 실행 → `downloads/`에 주차별 xlsx 추가
2. `streamlit run dashboard_app.py` 실행
3. 사이드바에서 비교 기준 주차 선택 → 탭별 분석 확인

## 3. 데이터 소스

### 3.1 입력 파일 규칙
- 위치: [downloads/](downloads/)
- 명명: `sales-{YYMMDD}-{YYMMDD}.xlsx`, `marketing-{YYMMDD}-{YYMMDD}.xlsx`
- 적재: 폴더의 **모든 파일을 자동 스캔**, 파일명에서 기간 파싱 → 주차 인덱스 생성

### 3.2 판매분석 (`상품성과` 시트, 22컬럼)
사용 컬럼:
- E `상품명` (전처리된 시리즈명: T60, T50, 아이블 …)
- A~D 카테고리(대/중/소/세)
- G `결제수`, H `결제상품수량`, J `결제금액`
- **U `실제수량` = H − R**, **V `실제금액` = J − P** (이미 가공됨, 환불 차감 반영)
- 보조 시트 `피벗_시리즈` (시리즈 / 실제수량 / 실제금액)
- **환불 관련 컬럼(O·P·R)은 대시보드에서 표시하지 않음** (환불일 기준이라 결제 주차와 어긋남)

### 3.3 마케팅분석 (`전체채널` 시트, 19컬럼)
사용 컬럼:
- A `채널속성` (모바일/PC), B `채널그룹` (광고/일반유입), C `채널명`, D `채널상세`
- E `고객수`, F `유입수`, **G `광고비`**
- J `결제수(마지막클릭)`, L `결제금액(마지막클릭)`, N `ROAS(마지막클릭)`
- O~S `+14일 기여도추정` 지표군
- 보조 시트 `피벗_채널` (채널명 / 고객수 / 유입수 / **광고비** / 결제금액)
  - ※ `create_marketing_pivot()` 수정 필요 — 광고비 합계 컬럼 추가

### 3.4 범위 외 (v1 제외)
- `수기파일.xlsx` — v2에서 통합 검토
- 환불 분석 (환불일 기준 데이터라 별도 분석 필요)
- 실시간 API 연동

## 4. 화면 구성

Streamlit 단일 앱, **사이드바 + 4개 탭** 구조.

### 4.1 사이드바
- **기준 주차** 셀렉트 (가장 최근 주가 기본값)
- **비교 주차** 셀렉트 (직전 주가 기본값, "비교 안함" 옵션)
- 시리즈 다중필터 (전체/T60/T50/…)
- 채널그룹 다중필터 (광고/일반유입/…)

### 4.2 탭 1 — 요약 (Overview)
**상단 KPI 카드 (7개), 기준 주차 vs 비교 주차 델타 표시**

| 카드 | 산식 | 출처 |
|---|---|---|
| 총 실제 매출 | Σ V열 | sales |
| 총 실제 수량 | Σ U열 | sales |
| 총 결제수 | Σ G열 | sales |
| 평균 객단가 | 실제매출 / 결제수 | sales |
| 총 유입수 | Σ F열 | marketing |
| 총 광고비 | Σ G열 | marketing |
| 통합 ROAS | 실제매출 / 광고비 | sales+mkt |

하단: 주차별 매출·유입수 듀얼 라인 차트 (전체 누적 주차)

### 4.3 탭 2 — 시리즈 매출 성과
- **시리즈별 막대차트**: 실제금액 (기준 주차, 정렬: 내림차순)
- **시리즈 × 주차 히트맵**: 셀 색 = 실제금액, hover = 실제수량/객단가
- **WoW 변동 테이블**: 시리즈, 이번주 매출, 지난주 매출, Δ금액, Δ%, 실제수량
- **카테고리(대/중/소) 트리맵** — 매출 비중 시각화

### 4.4 탭 3 — 마케팅 채널 효율
- **채널그룹별 ROAS 막대차트** (마지막클릭 vs +14일 기여도 비교)
- **채널 산점도**: x=유입수, y=결제금액, 점 크기=광고비, 색=채널그룹
- **상위 채널 테이블**: 채널명 / 유입수 / 광고비 / 결제금액 / ROAS / 유입당 결제율
  - 유입수가 있으나 ROAS가 0인 "비효율 채널" 강조 표시
- **광고비 vs 매출 추이** (주차별 듀얼축 라인)

### 4.5 탭 4 — 통합 분석 (Sales × Marketing)
> **주의:** 스마트스토어 데이터에서 채널과 상품을 직접 join할 키는 없음. 기간 단위 매크로 비교만 가능.

- **주차별 매트릭스**: 행=주차, 열=[실제매출, 광고비, ROAS, 유입수, 결제전환율, 객단가]
- **ROAS vs 매출 성장률** 스캐터 (각 점=1주차) — 광고 효율과 매출 동반 성장 주차 식별
- **채널 믹스 변화 영역차트**: 주차별 채널그룹 비중(광고/일반유입/SNS/검색)
- 시리즈별 "지난주 대비 매출 급증/급감" Top3 + 같은 주 상위 채널 변화 병기

## 5. 비기능 요구사항

| 항목 | 기준 |
|---|---|
| 성능 | downloads 30개 파일 기준 초기 로드 ≤ 3초 (`@st.cache_data` 사용) |
| 호환 | 기존 가공 로직(`process_excel` / `create_pivot`) 결과를 그대로 읽음, 수정 불요 |
| 환경 | Windows 11, Python 3.11+, 로컬 실행 (배포 X) |
| 의존성 | `streamlit`, `pandas`, `plotly`, `openpyxl` |
| 운영 | 새 주차 파일이 추가되면 별도 작업 없이 자동 인식 |
